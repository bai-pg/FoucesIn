# iPhone X 移动端优化 - 快速测试指南

## 🚀 立即测试

### 1. 启动开发服务器
```bash
npm run dev
```

### 2. Chrome DevTools 测试

#### 步骤：
1. **打开浏览器**，访问 `http://localhost:3000`（或你的开发端口）
2. **按 F12** 打开开发者工具
3. **按 Ctrl+Shift+M**（或点击设备工具栏图标）切换到设备模式
4. **选择设备**：
   - 顶部下拉菜单选择 "iPhone X"
   - 或手动设置：宽度 375px，高度 812px

#### 检查项：
- [ ] 页面加载后，主要内容可见（无需滚动）
- [ ] 顶部导航栏紧凑（~48px 高度）
- [ ] 标题字体适中（~20-22px）
- [ ] 卡片间距合理（~10-12px）
- [ ] 底部页脚不占用过多空间（~40px）
- [ ] 可以流畅滚动查看所有内容

### 3. 真机测试（推荐）

#### iOS Safari 测试：
1. 确保手机和电脑在同一 WiFi 网络
2. 在手机 Safari 中访问：`http://你的电脑IP:3000`
3. 检查显示效果

#### Android Chrome 测试：
1. 同样方式访问
2. 对比显示效果

## 🐛 常见问题排查

### Q1: 内容还是显示不完整？

**检查**：
1. 确认浏览器缓存已清除
2. 确认开发服务器已重启
3. 检查是否有自定义样式覆盖

**解决**：
```bash
# 清除缓存并重启
rm -rf node_modules/.vite
npm run dev
```

### Q2: 某些组件仍然过大？

**检查**：
- 该组件是否有固定的高度/宽度
- 是否有 inline style 覆盖
- Element Plus 组件是否有默认样式

**解决**：
在组件中添加移动端样式：
```vue
<style scoped>
@media (max-width: 375px) {
  .your-component {
    padding: 10px !important;
    font-size: 14px !important;
  }
}
</style>
```

### Q3: 滚动不流畅？

**检查**：
- 是否启用了硬件加速
- 是否有大量 DOM 节点
- 是否有复杂的动画

**解决**：
```css
.scroll-container {
  -webkit-overflow-scrolling: touch;
  will-change: transform;
}
```

### Q4: 安全区域未生效？

**检查**：
- meta viewport 是否包含 `viewport-fit=cover`
- 是否使用了 `.safe-area-*` 类

**解决**：
```html
<!-- index.html -->
<meta name="viewport" content="..., viewport-fit=cover" />
```

```vue
<!-- 使用安全区域类 -->
<header class="safe-area-top">...</header>
<footer class="safe-area-bottom">...</footer>
```

## 🎯 性能检查

### Lighthouse 评分
在 Chrome DevTools 中运行 Lighthouse：

1. 切换到 "Lighthouse" 标签
2. 选择 "Mobile" 设备
3. 点击 "Analyze page load"
4. 检查评分：
   - Performance: ≥ 90 ✅
   - Accessibility: ≥ 90 ✅
   - Best Practices: ≥ 90 ✅

### 关键指标
- **FCP** (First Contentful Paint): < 1.8s
- **LCP** (Largest Contentful Paint): < 2.5s
- **FID** (First Input Delay): < 100ms
- **CLS** (Cumulative Layout Shift): < 0.1
