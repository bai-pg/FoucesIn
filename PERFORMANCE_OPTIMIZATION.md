## ⚠️ 注意事项

### 1. 缓存策略调整

如需调整缓存时长,修改以下常量:

```typescript
// router.ts - 用户认证缓存
const CACHE_DURATION = 5 * 60 * 1000; // 5分钟

// StatsDashboard.vue - 统计数据缓存
const CACHE_DURATION = 2 * 60 * 1000; // 2分钟

// useGoals.ts - 目标数据缓存
const CACHE_DURATION = 5 * 60 * 1000; // 5分钟
```

### 2. 清理缓存

用户登出或需要强制刷新时:

```javascript
// 清除所有缓存
localStorage.clear();

// 或清除特定缓存
localStorage.removeItem('stats_userId');
localStorage.removeItem('goal_userId_weekly');
```

### 3. TypeScript 警告

`vite.config.ts` 中可能出现 TypeScript 类型检查警告,这是误报,不影响实际运行。如需消除警告,可以:

```typescript
// 在 manualChunks 函数上方添加注释忽略检查
// @ts-ignore
manualChunks(id) { ... }
```

## 📝 技术细节

### 分包策略原理

```
优化前:
├── index.js (2.5MB) - 所有代码打包在一起

优化后:
├── index.js (200KB) - 业务代码
├── vue-vendor.js (150KB) - Vue 核心
├── element-plus.js (400KB) - UI 库
├── echarts-vendor.js (500KB) - 图表库
├── supabase-vendor.js (100KB) - Supabase
└── x6-vendor.js (300KB) - 工作流库
```

**优势:**
- 浏览器可并行下载多个小文件
- 未修改的 chunk 可利用浏览器缓存
- 按需加载,减少初始下载量

### 缓存策略设计

```
首次访问:
用户 → 网络请求 → Supabase → 返回数据 → 存入 localStorage → 渲染页面

再次访问(缓存有效期内):
用户 → 读取 localStorage → 直接渲染页面 (无需网络请求)

缓存过期:
用户 → 网络请求 → 更新缓存 → 渲染页面
```

**优势:**
- 大幅减少网络请求次数
- 提升用户体验流畅度
- 降低服务器负载
